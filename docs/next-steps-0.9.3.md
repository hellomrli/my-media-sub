# 下一步更新建议报告

生成时间：2026-06-25

## 当前定位

项目应继续定位为“Rust 编写、夸克网盘专精的追剧自动化平台”。参考 `cloud-auto-save-x` 时，只借鉴产品结构和用户体验，不迁移 Python/FastAPI 技术栈，也不把多网盘作为主线。

## 本次已完成

- 将“转存历史”重构为“后台日志”，以后台任务为主数据源。
- 通知中心与后台执行记录完成职责拆分，避免同一转存结果在两个模块重复出现。
- 后台日志增加类型、状态、关键词筛选和任务详情弹窗，支持复制任务详情。
- 订阅创建弹窗升级为内容、规则、下载三步向导。
- 新增常用命名规则预设，可在订阅创建中快速套用。
- 系统设置新增“规则中心”，支持调试模板、过滤条件、同集保留策略和样例文件。
- 夸克设置新增账号健康面板，展示 Cookie 检测、自动转存、签到结果、连签进度和容量。

## 建议优先级

### P0：夸克追剧闭环继续打磨

- 把“规则中心”的规则预设持久化为用户可保存的多套方案。
- 为每个订阅增加“使用规则方案”字段，创建订阅时可选择方案而不是复制模板。
- 后台 Job 增加 `stage` 和 `run_log` 字段，保留更完整的执行阶段日志。
- 订阅检查结果持久化为最近 N 次记录，方便排查换源、跳集、误识别。

### P1：工作台

- 首页展示今日状态：追更中订阅、最近成功、最近失败、待处理后台任务、夸克健康、Aria2 状态。
- 展示“落后集数”和“最近更新订阅”，优先帮助判断今天是否需要处理。
- 将失败任务、失效链接、Cookie 异常聚合成待办列表。

### P1：追剧日历

- 基于 TMDB 下一集播出信息展示今天、本周和本月更新。
- 与订阅当前集数对比，显示已追平、落后、未知。
- 点击日历条目可进入订阅详情或立即检查。

### P2：夸克专精能力

- 增加夸克 Cookie 到期/失效风险提示。
- 增加夸克目录容量、目录文件数和最近变动摘要。
- 针对夸克分享链接增加更强的失效缓存、自动重试和换源辅助。

### P2：后处理插件，但保持 Rust 内核

- 可以做 Rust trait 风格的后处理 Hook：转存后触发 STRM、Aria2、媒体库刷新。
- 先内置少量稳定插件，不做多网盘插件市场。
- 插件配置仍写入现有设置或独立 JSON，避免引入复杂数据库迁移。

## 不建议近期投入

- 不建议转向多网盘账号体系，容易分散夸克追剧体验。
- 不建议引入 Python 服务或独立前端构建体系，当前 Rust 单体部署对 VPS 更简单。
- 不建议过早做多用户权限系统，除非后续明确多人共用场景。
