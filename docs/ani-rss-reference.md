# ani-rss 订阅与界面参考记录

参考项目：https://github.com/wushuo894/ani-rss

## 值得吸收的订阅逻辑

- 订阅卡片化展示：封面、标题、季数、启用状态、来源/字幕组、当前集数/总集数、最后下载时间。
- 列表可按星期分组，也可平铺，并支持搜索标题/拼音/首字母。
- 新增订阅支持多来源入口，避免用户手动找 RSS。对应到本项目：PanSou 搜索结果、手动分享链接、未来可接资源站/RSS。
- 不支持聚合订阅，避免一次过多更新导致遗漏。对应到本项目：订阅应绑定到具体分享目录，而不是大量混合搜索结果。
- 单订阅字段丰富：季数、发布时间、星期、字幕组、匹配词、排除词、启用状态、当前集数、总集数、自定义下载位置、自定义集数识别、只下载最新集、跳过集数、通知开关、完结状态。
- 任务执行有全局开关、检查间隔、互斥锁、每个订阅之间延迟，避免频繁请求和风控。
- 日志页支持级别筛选、logger 筛选、下载日志、刷新、清空。
- 下载器抽象支持 Aria2/qBittorrent/Transmission/OpenList。
- 通知抽象支持系统通知、Telegram、Webhook、邮件、ServerChan 等。

## 本项目可迁移设计

### 已实现

- 订阅卡片
- 订阅启用/失效状态
- 当前文件数、新文件通知
- 失效链接通知
- 搜索/订阅/日志下载/设置分页
- Aria2 基础提交

### 下一步优先级

1. 订阅字段增强：季数、总集数、当前集数、更新星期、来源组、封面、匹配词/排除词、只追最新。
2. 自动检查任务：全局检查间隔、检查延迟、互斥执行、避免风控。
3. 下载策略：发现新文件后询问/自动下载；支持只下载最新集。
4. 日志系统：结构化日志、级别筛选、清空/下载。
5. 订阅卡片优化：封面、进度标签、最后更新时间、启用开关。
6. 外部通知：微信/企业微信/Webhook/Telegram。

## 注意

ani-rss 面向 RSS/torrent，本项目面向网盘分享目录。不能照搬 RSS 聚合/下载逻辑；应保留“订阅具体目录、低频检查、发现新增文件后通知或转存”的模型。
