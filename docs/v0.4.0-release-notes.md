# v0.4.0 重大更新说明

## 🎯 本次更新内容

本次更新是一次重大架构升级，包含以下核心改进：

### 1. ⚡ 异步化改造
- 所有网络请求从 `requests` 迁移到 `httpx.AsyncClient`
- 搜索聚合并发执行，响应时间从 30s+ 降至 5-10s
- 订阅检查并发执行，100个订阅从串行 10+ 分钟降至并发 1-2 分钟
- 新增重试机制（`tenacity`），提升稳定性

### 2. 🔄 后台任务队列
- 基于 `asyncio.Queue` 的轻量任务队列
- 支持任务优先级、并发控制、失败重试
- 新增 `/api/queue/status` 接口查看队列状态
- 订阅检查、文件转存等耗时操作全部异步化

### 3. 🗄️ 数据持久化升级
- 从 JSON 文件迁移到 **SQLite + SQLAlchemy**
- 新增 3 张表：
  - `subscriptions` - 订阅记录
  - `notifications` - 通知记录
  - `transfer_records` - 转存记录
- 支持复杂查询、事务、并发安全
- 提供数据迁移脚本

### 4. ⚙️ 配置管理优化
- 统一使用 **Pydantic BaseSettings**
- `.env` 文件优先级：环境变量 > 文件 > 默认值
- 废弃 `settings.json`，全部配置在 `.env`
- 新增配置项：
  ```env
  DATABASE_URL=sqlite+aiosqlite:///./data/app.db
  TELEGRAM_BOT_TOKEN=your_token
  TELEGRAM_CHAT_ID=your_chat_id
  NOTIFICATION_LEVEL=info
  PREFERRED_QUALITY_ORDER=4K,2160p,1080p,720p,480p
  AUTO_COMPLETE_AFTER_NO_UPDATES=5
  ```

### 5. 🎨 智能去重
- 基于标题相似度（`difflib.SequenceMatcher`）去重
- 自动提取画质标识（4K/1080P/720P）
- 根据配置的画质偏好选择最佳版本
- 搜索结果新增 `quality` 和 `quality_rank` 字段

### 6. 📱 Telegram 通知
- 集成 `python-telegram-bot`
- 支持订阅更新、检查失败、自动完结等通知
- 可配置通知级别（info/success/warning/error）
- 消息格式化（HTML + Emoji）

### 7. 🏁 订阅自动完结
- 连续 N 次（可配置）检查无新增内容 → 自动标记完结
- 剧集数达到预期 → 自动标记完结
- 完结时发送 Telegram 通知
- 避免无效订阅持续占用资源

### 8. 🛠️ 代码规范
- 引入 **ruff** 代码格式化和 lint
- 引入 **mypy** 类型检查
- 配置 **pre-commit hooks** 自动检查
- 统一代码风格，提升可维护性

## 📊 性能对比

| 检查 50 个订阅 | ~5min | ~1min | **5x** |

## 🚀 如何升级

详见 [UPGRADE.md](./UPGRADE.md)

简要步骤：
1. `pip install -r requirements.txt`
2. `python -m src.scripts.migrate_to_db`
3. 更新 `.env` 配置
4. 重启服务

## ⚠️ 破坏性变更

1. **Python 版本要求** - 最低 Python 3.11
2. **数据存储** - JSON → SQLite（需要迁移）
3. **配置文件** - `settings.json` → `.env`（需要迁移）
4. **导入路径** - 部分模块路径调整
