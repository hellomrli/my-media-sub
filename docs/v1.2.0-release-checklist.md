## 2. 拆分提交安全规则

- 不使用 `git reset --hard`；
- 不覆盖或丢弃当前未提交工作；
- 每个提交前保存 `git diff --binary` 补丁备份；
- 对交叉文件使用 `git add -p`；
- 每个逻辑提交后至少运行相关测试；
- 全部提交完成后再运行完整质量门；
- 不在未获授权时执行 commit、tag、push 或创建 Release。

建议在开始拆分前：

```bash
git diff --binary > /tmp/my-media-sub-v1.2.0-worktree.patch
git status --short
git diff --check
```
