# v2.2.11 升级指南

```bash
docker compose pull
docker compose up -d
docker compose logs --tail=100 -f
```

v2.2.11 不修改 JSON Store schema，可直接复用 v2.2.10 的 `data/`。

本版本以可靠性修复为主：转存业务级幂等（重试不再重复转存）、消除三处并发写回丢更新、批量检查互斥、电影订阅补转、分享探测分页拉全并显式标记截断（上限 200 → 500）。另外整理了订阅列表界面（双视图共用操作菜单、顶部面板合并）。前端资源 URL 带有 `?v=2.2.11`。

行为变化提示：

- 已有批量检查在跑时，再次触发批量检查会返回 400「批量检查已在进行中」，稍后重试即可。
- 若历史上曾有数据文件损坏被隔离（`data/*.json.corrupt-*`），升级后启动会收到一条站内告警通知，请按提示核对备份。

如果使用二进制部署，必须同时替换归档中的整个 `static/` 目录；只替换二进制会继续运行旧前端代码。
